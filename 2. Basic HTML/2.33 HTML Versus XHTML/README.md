## HTML5 сейчас

```html
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Страница</title>
</head>
<body>
    <p>Контент</p>
</body>
</html>
```

- Доктайп короткий: `<!DOCTYPE html>`
- Браузер **восстанавливает** мелкие ошибки разметки (незакрытые теги) — но на это полагаться нельзя
- Расширение `.html`

## Что было XHTML

**XHTML** (2000-е) — HTML, записанный по правилам XML:

- Все теги закрыты (`<br />`)
- Строчный регистр
- Атрибуты в кавычках
- Ошибка парсера могла остановить отображение

Идея — единый строгий синтаксис. На практике победил **HTML5** с более мягким парсером и новыми семантическими тегами.

| | HTML5 | XHTML |
|---|-------|-------|
| Актуальность | ✅ стандарт | исторический |
| Doctype | `<!DOCTYPE html>` | длинный XML doctype |
| Пустые теги | `<br>`, `<img>` | `<br />` |
| Ошибки | браузер чинит | мог не показать страницу |

## Строгие привычки

Пишите разметку так, будто парсер строгий — это и есть современная практика:

1. Закрывайте парные теги
2. Вложенность в правильном порядке
3. `lang`, `charset`, `title`, `alt`
4. Семантические теги вместо «супа из div»
5. Валидатор: [validator.w3.org](https://validator.w3.org/)

Это даёт пользу XHTML без его формальностей.
