## Зачем

В URL разрешён ограниченный набор символов. Пробелы, кириллица, `&`, `?`, `#` в **значениях параметров** кодируют, иначе адрес разберётся неверно.

Это **не то же самое**, что HTML-сущности (`&lt;`) — другой контекст.

## Правила

| Символ | Кодировка |
|--------|-----------|
| пробел | `%20` или `+` в query form-данных |
| `&` | `%26` |
| `=` | `%3D` |
| `?` | `%3F` |
| кириллица | байты UTF-8 в `%D0%9F…` |

```
https://example.com/search?q=%D0%BF%D1%80%D0%B8%D0%B2%D0%B5%D1%82
```

## Примеры

Ссылка с параметром:

```html
<a href="https://example.com/search?q=hello%20world">Поиск</a>
```

Форма с `method="get"` кодирует поля автоматически.

## JavaScript

```javascript
encodeURIComponent('привет мир');  // для части query
decodeURIComponent('%D0%BF%D1%80%D0%B8%D0%B2%D0%B5%D1%82');

encodeURI('https://example.com/путь');  // целый URL
```

`encodeURIComponent` кодирует больше символов — используйте для значений параметров.
