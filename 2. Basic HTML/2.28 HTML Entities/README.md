## Зачем

Символы `<`, `>`, `&`, кавычки в **тексте** могут сломать парсер или смешаться с тегами. В UTF-8 многие символы можно писать напрямую, но сущности всё ещё нужны для `<>&"'` в HTML-тексте.

## Синтаксис

```
&имя;
&#число;
&#xHEX;
```

```html
<p>5 &lt; 10 и &quot;цитата&quot;</p>
<p>&copy; 2026 Компания</p>
<p>&#8364; — евро через номер</p>
```

## Частые сущности

| Символ | Имя | Номер |
|--------|-----|-------|
| `<` | `&lt;` | `&#60;` |
| `>` | `&gt;` | `&#62;` |
| `&` | `&amp;` | `&#38;` |
| `"` | `&quot;` | `&#34;` |
| `'` | `&apos;` | `&#39;` |
| `©` | `&copy;` | `&#169;` |
| `®` | `&reg;` | `&#174;` |
| `€` | `&euro;` | `&#8364;` |

В **атрибутах** с двойными кавычками внутренние `"` экранируйте или используйте одинарные кавычки снаружи.

## Неразрывный пробел

`&nbsp;` — пробел без переноса строки. Удобно для «10 кг», «стр. 5», но не злоупотребляйте вместо нормального CSS.

Лишние пробелы в HTML и так схлопываются; для отступов — CSS `margin`/`padding`.
