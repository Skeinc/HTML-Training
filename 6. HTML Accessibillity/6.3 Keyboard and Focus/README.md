# Клавиатура и фокус

Многие пользователи не пользуются мышью: клавиатура, переключатели, голосовое управление. Страница должна быть **полностью проходимой** с Tab.

## Базовая навигация

| Клавиша | Действие |
|---------|----------|
| **Tab** | Следующий фокусируемый элемент |
| **Shift + Tab** | Предыдущий |
| **Enter** | Активировать ссылку или кнопку |
| **Space** | Кнопка, checkbox |
| **Esc** | Закрыть `<dialog>`, выпадающее меню |

Фокусируемые по умолчанию: ссылки, кнопки, поля форм, `tabindex="0"`.

## Skip link

Первая ссылка на странице — «Перейти к содержимому», чтобы не табать через всё меню:

```html
<a href="#main" class="skip-link">Перейти к содержимому</a>
…
<main id="main">…</main>
```

В CSS ссылку показывают при фокусе (иначе скрыта визуально, но доступна с клавиатуры).

## Видимый фокус

Не убирайте outline без замены:

```css
/* плохо */
*:focus { outline: none; }

/* лучше */
:focus-visible {
    outline: 2px solid #2563eb;
    outline-offset: 2px;
}
```

`:focus-visible` — обводка только при навигации с клавиатуры, не при клике мышью.

## `tabindex`

| Значение | Поведение |
|----------|-----------|
| нет атрибута | Естественный порядок (ссылки, кнопки, поля) |
| `0` | Элемент в Tab-порядке (для кастомных виджетов) |
| `-1` | Фокус программно (`element.focus()`), не в Tab |
| `> 0` | **Избегать** — ломает логичный порядок |

## Модальные окна

Нативный [`<dialog>`](../../2.%20Basic%20HTML/2.34%20HTML%20Dialog/README.md) с `showModal()`:
- переносит фокус внутрь;
- ловит Tab внутри диалога;
- Esc закрывает.

Для кастомных модалок на `div` нужны ARIA (`role="dialog"`, `aria-modal="true"`) и ручной focus trap — сложнее, чем `<dialog>`.

Пример skip link и фокуса — в [`index.html`](index.html).
