## Принципы POUR

| Принцип | Суть |
|---------|------|
| **Perceivable** | Контент воспринимается разными органами чувств (текст вместо картинки, субтитры) |
| **Operable** | Можно пользоваться с клавиатуры, без ловушек фокуса |
| **Understandable** | Понятный язык, предсказуемое поведение |
| **Robust** | Корректная разметка для браузеров и ассистивных технологий |

## HTML на практике

**Семантика** — не только `div`:

```html
<header>…</header>
<nav aria-label="Главное меню">…</nav>
<main>…</main>
<footer>…</footer>
```

**Изображения** — осмысленный `alt`; декоративным `alt=""`.

**Формы** — `<label for="id">`, группы `<fieldset>`/`<legend>`, `required`, сообщения об ошибках.

**Ссылки и кнопки** — понятный текст («Скачать отчёт», не «нажмите здесь»).

**Язык** — `<html lang="ru">`, `lang` на фрагментах иностранного текста.

**Заголовки** — логичная иерархия `h1`–`h6`, один `h1` на страницу.

**Медиа** — `<track>` для субтитров; не автозвук.

**Фокус** — видимый `:focus` в CSS; порядок табуляции логичен.

## Чеклист

- [ ] У каждого `<img>` есть подходящий `alt`
- [ ] У форм есть labels, ошибки не только цветом
- [ ] Контраст текста ≥ 4.5:1 (обычный текст)
- [ ] Страница проходима с Tab без мыши
- [ ] `title` у iframe, `aria-label` у icon-only кнопок
- [ ] Нет блокировки zoom (`user-scalable=no` в viewport)

## Инструменты

- Lighthouse (Chrome DevTools) — аудит a11y
- axe DevTools — расширение браузера
- Скринридер: VoiceOver (macOS), NVDA (Windows)
- Клавиатура: пройдите сайт только Tab / Enter / Esc
