# HTML-атрибуты

Атрибуты дополняют элемент: задают ссылку, источник картинки, имя поля формы, класс для CSS и т.д. Указываются **только в открывающем теге**.

## Синтаксис

```html
<элемент атрибут="значение" другой="значение2">…</элемент>
```

| Правило | Пояснение |
|---------|-----------|
| Формат | `имя="значение"` (пара имя/значение) |
| Где писать | В открывающем теге |
| Кавычки | Значения всегда в кавычках (`"` или `'`) |
| Регистр имён | Строчные буквы: `class`, не `CLASS` |

**Булевы атрибуты** — имя без значения означает «включено»:

```html
<input type="text" disabled>
<input type="text" disabled="disabled">  <!-- то же самое -->
```

## Глобальные и локальные

**Глобальные** — работают на любом элементе:

| Атрибут | Назначение |
|---------|------------|
| `id` | Уникальный идентификатор на странице |
| `class` | Имя для CSS и выборки в JS |
| `style` | Инлайн-стили (лучше избегать в пользу CSS-файлов) |
| `lang` | Язык фрагмента (`lang="en"`) |
| `hidden` | Скрыть элемент |
| `title` | Всплывающая подсказка |

**Локальные** — только у конкретных тегов: `href` у `<a>`, `src` и `alt` у `<img>`, `type` у `<input>`.

Полный список — в [справочнике глобальных атрибутов](../../HTML%20Reference/HTML%20Global%20Attributes/README.md).

## Примеры по задачам

**Ссылка и изображение:**

```html
<a href="https://example.com">Перейти</a>
<img src="cat.jpg" alt="Рыжий кот на подоконнике">
```

`src` у `<img>` обязателен для отображения. `alt` формально необязателен, но **нужен для доступности** — без него скринридеры и поисковики теряют смысл картинки.

**Семантика и доступность:**

```html
<nav aria-label="Главное меню">…</nav>
<button type="button" aria-expanded="false">Меню</button>
```

`role` и `aria-*` помогают ассистивным технологиям, когда нативной семантики тега недостаточно.

**Формы** (подробнее — в разделе [HTML Forms](../../3.%20HTML%20Forms/README.md)):

```html
<form action="/search" method="get">
    <input type="search" name="q" placeholder="Поиск">
</form>
```

**События** (`onclick`, `onchange` и др.) — допустимы в HTML, но в современных проектах обработчики обычно вешают из JavaScript (`addEventListener`), а не атрибутами.

## Правила записи

1. **Имена атрибутов** — строчными буквами.
2. **Значения** — всегда в кавычках.
3. **Одинарные или двойные кавычки** — без разницы; если внутри значения есть `"`, оберните в `'`, и наоборот:

```html
<p title='Он сказал: "Привет"'>…</p>
```
