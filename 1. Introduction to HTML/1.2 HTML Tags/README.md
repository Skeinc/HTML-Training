# HTML-теги

Теги — строительные блоки разметки. Браузер по тегам понимает, что перед ним заголовок, абзац, ссылка или изображение.

## Синтаксис

**Парный элемент** — открывающий тег, содержимое, закрывающий тег:

```html
<p>Текст абзаца</p>
```

**Структура тега:**

```
<имя атрибут="значение">содержимое</имя>
```

| Часть | Пример |
|-------|--------|
| Открывающий тег | `<h1>` |
| Содержимое | `Заголовок` |
| Закрывающий тег | `</h1>` |

Имя в закрывающем теге **должно совпадать** с открывающим: `<p>…</p>`, не `</P>`.

## Парные и пустые элементы

**Пустые (void) элементы** не имеют содержимого и закрывающего тега — например `<br>`, `<img>`, `<input>`, `<meta>`.

```html
<p>Первая строка<br>Вторая строка</p>
<img src="photo.jpg" alt="Фото">
```

В HTML5 слэш в конце (`<br />`) не обязателен.

| Тип | Примеры | Закрывающий тег |
|-----|---------|-----------------|
| Парные | `<p>`, `<a>`, `<div>`, `<h1>`–`<h6>` | Обязателен |
| Пустые | `<br>`, `<img>`, `<hr>`, `<input>` | Не используется |

Браузер иногда «исправляет» незакрытые теги, но на это **полагаться нельзя** — разметка может сломаться непредсказуемо.

## Вложенность

Элементы вкладываются друг в друга. Теги должны закрываться **в обратном порядке**:

```html
<!-- верно -->
<p>Текст с <strong>важным</strong> словом</p>

<!-- неверно -->
<p>Текст с <strong>важным</p> словом</strong>
```

## Регистр и стиль кода

HTML **не чувствителен к регистру** тегов: `<P>` и `<p>` эквивалентны. На практике всегда пишите теги **строчными** — так принято в HTML5, в редакторах и в чужом коде.
